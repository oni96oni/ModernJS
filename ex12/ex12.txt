함수는 객체타입의 값이다.
    값 : 표현식이 평가되어 생성된 결과

함수는 함수 이름으로 호출하는 것이 아니라 함수 객체를 가리키는 식별자로 호출한다.
    이때, 식별자는 자바스크립트엔진에서 함수 이름과 동일한 이름의 식별자를 암묵적으로 생성, 거기에 함수 객체를 할당.

변수 호이스팅 : undefined로 초기화
함수 호이스팅 : 함수 객체로 초기화

함수 호출시 초과된 인수는 무시된다.
    버려지는것이 아니라 암묵적으로 arguments 객체의 프로퍼티로 보관된다.

매개변수의 개수가 많다 -> 함수가 여러가지 일을 한다. -> not good!
3개 이상의 매개변수가 필요하다면 하나의 매개변수를 선언하고 객체를 인수로 전달하는것이 유리하다.

return; 은 undefined를 반환한다.

원시타입인수는 원본 훼손 x
객체타입인수는 원본 훼손 o

즉시 실행 함수는 반드시 그룹연산자 ( ) 로 감싸야 한다
    왜? 자스엔진이 함수 호출 연산자가 아니라 그룹연산자로 해석해서 에러발생



비순수함수, 조건문, 반복문 보다는 보조함수를 이용하는게 바람직!
